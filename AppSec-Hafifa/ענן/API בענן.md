# **API בענן**

**APIים בענן הציבורי הם מה שמאפשרים תקשורת לענן והחוצה ממנו , ה-API מאפשר תקשורת בין שירותי מחשוב ענן שונים או בין שירותי הענן לאפליקציות מקומיות.**

**API בענן מאפשרות לפתח אפליקציה או שירותים בענן עצמו , בנוסף APIs מאפשרים לחבר בין כמה עננים (גם מסוגים שונים כמו on prem) ולאפשר למפתחים עבודה עם משאבים מעבר למשאבי הענן שנמצאים בארגון שלהם , APIs עוזרים לנהל את משאבי הענן בצורה יעילה ומסוגלים להשתלב בתוך סביבה מאובטחת.**

**בזכות שימוש בAPIs אפליקציות שונות יכולות לעבוד בצורה טובה יותר עם שירותי עננים שונים כמו אחסון או מסדי נתונים.**

**הצורה שבדרך כלל API מבצעים את התקשורת שלהם בין שירותי ענן או בין הענן לאפליקצה היא בעזרת API CLIENT ו API ENDPOINT (אחד נמצא על צד הלקוח והשני על השרת) הלקוח צריך לבצע פעולה או לקבל מידע הוא מבצע API CALL הAPI בשרת מקבלת את הבקשה מוודא שהבקשה היא לגיטימית מבחינת שולח ופורט פרוטוקול API נכון של הבקשה ומחזירה את מה שהלקוח ביקש.**

**בעזרת ה API ניתנת האפשרות ליצור את השירותים (SAAS ,PAAS,IAAS) שענן מספק , API לתשתית עוזר לנהל ולתכנן את התשתית בענן , בעוד שAPI של פלטפורמה או אפליקציה מקשר את התשתית לגורם צד שלישי שמספק את הפלטפורמה או האפליקציה.**

**IAAS API שולט על 3 אזורים ספציפים : COMPUTE , STORAGE ,NETWORKING.**

**החלק של הCOMPUTE הוא מה שמאפשר הקמה של VIRTUAL MACHINE , בעזרת API הזה אפשר לנהל ולשלוט על VM ומופעים אחרים בשרת , כמו יצירה מחיקה התחלה ועצירה וכדומה , זה יקרה בצורה שבה API עובד בענן.**

- **הלקוח עושה API REQUEST עם הקמה של VM**
- **שרת הענן בודק שיש לו את ההרשאות , מוודא את הזהות שלו ובודק שיש לו את המשאבים בשביל להקים את הVM**
- **שרת הענן יוצר את הVM**

**נצטרך את הפרמטרים הבאים : SSH KEY מג'ונרט שנספק לענן , שם הVM , שם קבוצת המשאבים , image של הVM שנרצה להקים ושם המשתמש של הADMIN בVM הזה.**

*** API מאפשר פורנזיקה יותר טובה בענן הציבורי , עוזר באיסוף ארטיפקטים**
